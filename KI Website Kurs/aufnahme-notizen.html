Du sprichst frei – das hier ist dein Anker

Für jedes Video: kurze Stichpunkte was rein muss. Kein Skript, kein Ablesen. Nur damit du nichts vergisst.

Modul 01 Warum das alles verändert
1.1 Willkommen – Was dich erwartet ~3 Min
  • Kurz vorstellen: wer du bist, ein Satz
  • Was am Ende rauskommt: fertige, professionelle Website – ohne Code
  • Sechs Module, nach jedem ein konkretes Ergebnis
  • Ehrlichkeit: „Das hätte ich früher gebraucht"
  • Kein Vorwissen nötig – wenn du eine App öffnen kannst, reicht das
  • Abschluss mit Energie: „Fangen wir an."
1.2 Das Werkzeug, das alles ändert ~5 Min
  • Der Unterschied klar machen:
    • ChatGPT = erklärt dir wie man kocht
    • Claude Code = kocht für dich, räumt auf, kauft Zutaten
  • Claude Code schreibt, bearbeitet und speichert Dateien – selbstständig
  • Nicht nur antworten – es macht etwas auf deinem Computer
  • Analogie: ChatGPT = Taschenrechner / Claude Code = Buchhalter der alles für dich erledigt
  • Kurz zeigen: einen Befehl eingeben, Claude schreibt eine Datei – live
  • Takeaway: Das ist kein Hype. Das verändert wie du arbeitest.
Modul 02 Einrichtung – einmal, dann nie wieder
2.1 Claude Code installieren ~4 Min
  • claude.ai aufrufen → Download der Desktop App (nicht Terminal)
  • Installieren wie jede normale App
  • Beim ersten Start: einloggen oder Account anlegen
  • Zeigen: App ist offen, Interface sichtbar, alles bereit
  • Abschluss: „Das war's. Eine Installation."
2.2 So sieht dein Arbeitsplatz aus ~3 Min
  • Linkes Fenster: claude.ai im Browser – für Gespräche, Brainstorming, Recherche
  • Rechtes Fenster: Claude Code Desktop App – baut die Website
  • Warum zwei? Browser = denken / Desktop App = bauen
  • Das Zwei-Fenster-Setup kurz live zeigen
  • Takeaway: So arbeitest du ab jetzt immer
2.3 Desktop App vs. Terminal – Was du brauchst ~2 Min
  • Terminal = schwarzes Fenster voller Code → brauchst du NICHT
  • Desktop App = normales Chat-Interface, vertraut, einfach
  • Alle Videos hier nutzen nur die Desktop App
  • Terminal nur relevant wenn du Entwickler bist – das bist du nicht, und das ist gut so
2.4 Node.js installieren ~3 Min
  • nodejs.org aufrufen → LTS-Version herunterladen (linke Schaltfläche)
  • Installieren, alle Standardoptionen unverändert lassen
  • Warum: Claude Code braucht es im Hintergrund zum Ausführen
  • Du wirst es nie direkt öffnen oder benutzen
  • „Einmal installiert, für immer erledigt."
2.5 Windows oder Mac? ~3 Min
  • Mac-User: alles schon dabei – direkt weiter zu Video 2.6
  • Windows-User: kurz Git nachinstallieren:
    • git-scm.com aufrufen
    • Herunterladen und installieren – alle Standardoptionen lassen
    • Warum: Claude Code nutzt Git im Hintergrund
  • Danach: Windows und Mac identisch – kein Unterschied mehr
2.6 Erster Testrun ~3 Min
  • Claude Code Desktop App öffnen
  • Neues Projekt anlegen → Ordner auf dem Computer auswählen
  • Ersten Befehl eintippen: „Erstell eine einfache HTML-Datei mit Hallo Welt"
  • Claude schreibt – live zusehen
  • Ordner aufmachen: Datei ist da
  • Datei im Browser öffnen: es funktioniert
  • Takeaway: „Das war dein erster Schritt. So einfach ist das."
Modul 03 Jetzt geht's ans Eingemachte
3.1 Wie Claude Code wirklich denkt ~6 Min
  • Claude Code denkt in Projekten, nicht in Gesprächen
  • Modell wählen:
    • Sonnet = Standard, für normale Aufgaben
    • Opus = für komplexe Dinge (langsamer, teurer)
  • Projekt-Ordner anlegen: sauber strukturiert, ein Ordner pro Website
  • Memory-Datei (CLAUDE.md) einrichten – das ist entscheidend:
    • Claude vergisst bei jedem Start wer du bist
    • Mit Memory-Datei: weiß immer den Kontext, dein Stil, dein Projekt
    • Reinschreiben: wer du bist, was du baust, welchen Ton du willst
  • Live zeigen: Memory-Datei anlegen und füllen
3.2 Neue Website oder Überarbeitung? ~4 Min
  • Zwei Wege, beide funktionieren – du entscheidest dich jetzt:
  • Neu von Null:
    • Komplett frischer Start
    • Kein Altlast-Code, moderner Aufbau
    • Empfehlung für die meisten
  • Bestehende Website überarbeiten:
    • Code importieren, Claude modernisiert ihn
    • Struktur und Inhalte bleiben, Optik wird besser
    • Geeignet wenn du mit der Struktur zufrieden bist
  • Abschluss: „Entscheid dich jetzt für deinen Weg – dann starten wir."
3.3 Brand Guidelines mit KI entwickeln ~6 Min
  • Claude stellt Fragen – du beantwortest sie:
    • Wer bist du? Was machst du?
    • Wen willst du ansprechen?
    • Wie klingst du? Professionell, locker, direkt?
  • Aus deinen Antworten entwickelt Claude:
    • Farbpalette passend zu deiner Marke
    • Schrift-Empfehlungen
    • Tonalität mit Beispielsätzen
  • Ergebnis in Memory-Datei speichern → Claude nutzt das jetzt immer
  • Live zeigen mit fiktivem Beispiel-Profil
  • Takeaway: „Deine Marke ist jetzt in Worte gefasst. Das nutzen wir ab jetzt."
3.4 Die Website entsteht ~8 Min
  • Den kompletten Loop live zeigen:
    • Prompt eingeben: „Bau meine Homepage mit Header, Über-mich, Angeboten, Kontakt"
    • Claude schreibt den Code
    • Datei im Browser öffnen und anschauen
    • Feedback geben: „Mach die Überschrift größer, ändere die Farbe zu [X]"
    • Claude überarbeitet
  • Zeigen: 3–4 Iterationen bis es gut aussieht
  • Wichtig betonen: Beim ersten Mal ist es nie perfekt – das ist vollkommen normal
  • Iteration ist der Prozess, nicht der Fehler
  • Takeaway: „Deine Website existiert. Das war Modul 3."
Modul 04 Website live schalten
4.1 GitHub einrichten und verbinden ~5 Min
  • github.com → kostenlosen Account anlegen
  • Neues Repository erstellen:
    • Name: deine-website (Kleinbuchstaben, Bindestriche)
    • Sichtbarkeit: Public – damit GitHub Pages funktioniert
  • Claude Code mit GitHub verbinden (Befehl live zeigen)
  • Lokalen Website-Ordner mit dem Repository verbinden
  • Kurz erklären was ein Repository ist: „Dein Cloud-Ordner für den Code"
4.2 Änderungen pushen – deine Website ist live ~4 Min
  • Ersten Push durchführen (live zeigen)
  • GitHub Pages aktivieren: Settings → Pages → Branch: main
  • Warten (30–60 Sekunden) – URL erscheint
  • URL aufrufen: Website ist live im Internet
  • Das Moment genießen: „Das ist deine Website. Für jeden erreichbar."
  • Optional: eigene Domain verknüpfen – kurz zeigen wo das geht
  • Ab jetzt: jede Änderung einfach pushen → automatisch live
Modul 05 SEO – dein stiller Vertrieb
5.1 Technische SEO in 15 Minuten ~8 Min
  • Kein Theorie-Video – direkt Ergebnisse zeigen
  • Befehl an Claude: „Optimiere meine Website für SEO"
  • Was Claude automatisch macht:
    • Meta-Title und Meta-Description für jede Seite
    • Alt-Texte für alle Bilder
    • Überschriften-Struktur (H1, H2, H3) prüfen und korrigieren
    • Ladezeit optimieren
    • Mobile-Darstellung sicherstellen
  • Google Search Console kurz erwähnen: kostenlos, zeigt wie Google dich sieht
  • Takeaway: „Das war technische SEO. Erledigt in einem Prompt."
5.2 Blog als Kundenmagnet aufbauen ~6 Min
  • Warum Blog: Google liebt regelmäßigen Content → du wirst gefunden
  • Ohne Blog: Google kennt dich kaum. Mit Blog: Google empfiehlt dich
  • Blog-Struktur anlegen: Claude baut die Vorlage einmalig
  • Ersten Artikel schreiben:
    • Thema: häufigste Frage deiner Zielgruppe beantworten
    • Du gibst 3–5 Stichpunkte
    • Claude schreibt den Artikel – SEO-optimiert, fertig
  • Pushen → Artikel ist live
  • Aufwand dauerhaft: 30–60 Minuten pro Artikel – komplett mit KI
  • Takeaway: „Einmal aufsetzen. Dauerhaft profitieren."
Modul 06 Was jetzt kommt
6.1 Du hast eine Website. Was als nächstes? ~5 Min
  • Würdigung: Was du in diesem Kurs gebaut hast:
    • Eigene Website – ohne Programmierkenntnisse
    • Live im Internet, erreichbar für jeden
    • SEO-optimiert
    • Blog-fähig, wächst weiter
  • Das war KI als Werkzeug für eine konkrete Aufgabe
  • Wer KI wirklich als Business-Hebel einsetzen will:
    • Folgeprogramm: KI in alle Bereiche des Business bringen
    • 2.500 € · 10 Plätze (limitiert) · 1. Durchlauf · Raten möglich
    • Persönlich ansprechen – kein Druck, kein Pitch-Druck
  • CTA: „Wenn es sich für dich richtig anfühlt, schreib mir einfach."
  • Abschluss warm und authentisch – nicht verkäuferisch
